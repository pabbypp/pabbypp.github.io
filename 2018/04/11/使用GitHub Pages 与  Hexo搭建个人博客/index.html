使用GitHub Pages与Hexo搭建个人博客

  • 使用它们搭建博客的原因

Hexo 是用来生成 HTML 的，github pages 用来展示 HTML。

  • GitHub pages服务器托管

将项目源码托管在 github 上，使用 github 的 pages 服务。其实 github 的 pages 服务不只是可以展示博客，你的每一个 github 仓库都有 pages 服务，可以通过简单的设置通过项目的 index.html 为入口展示你的项目，这一点也很实用！但是大部分的 pages 服务都是用来搭建个人博客的.

第一步：新建一个git仓库，注意仓库名命名一定得是以下格式：用户名.github.io，这个仓库存放所有博客文章等。

  • 腾讯云购买域名

  1. 域名指向github的IP地址：

在腾讯云服务域名管理解析那添加github的IP地址

  1. github 绑定自己的域名：

a. 向你的 Github Pages 仓库添加一个CNAME(一定要大写)文件其中只能包含一个顶级域名，像这样：pabbypp.com
如果你是用 hexo 框架搭建博客并部署到 Github Pages 上，每次
hexo g && hexo d
后会把你的博客所在目录下 public 文件夹里的东西都推到 Github Pages 仓库上，并且把 CNAME 文件覆盖掉，解决这个问题可以直接把 CNAME 文件添加到 source 文件夹里，这样每次推的时候就不用担心仓库里的 CNAME 文件被覆盖掉了。

b. 向你的 DNS 配置中添加 3 条记录

@ A 192.30.252.153
@ A 192.30.252.154
@ CNAME pabbypp.github.io.

用你自己的 Github 用户名替换 username配置 DNS 推荐使用 DNSPOD 的服务，使用国外的 DNS 解析服务可能有被墙的风险。至于如何使用 DNSPOD 解析域名，参考 http://jingyan.baidu.com/article/546ae1857c4ee81149f28cbe.html

c. 等待你的 DNS 配置生效对DNS的配置不是立即生效的，过10分钟再去访问你的域名看看有没有配置成功 :

  • Hexo 搭建静态网页

  • 什么是Hexo？

    Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

  • 安装hexo

  1. 检查是否安装以下应用程序： 在命令行输入 npm -V

    • Node.js
    • Git

  2. 使用npm 即可完成Hexo的安装

    $ npm install -g hexo-cli

  3. 完成Hexo安装后，建站

    在电脑的某个地方新建一个名为hexo的文件夹（名字可以随便取），比如我的是F:\Workspaces\hexo，由于这个文件夹将来就作为你存放代码的地方，所以最好不要随便放。

    $ cd /f/Workspaces/hexo/
    $ hexo init

    （hexo会自动下载一些文件到这个目录，包括node_modules）

    $ hexo g # 生成
    $ hexo s # 启动服务

  4. Hexo文件夹下内容：


    hexo/
      |- node_modules/  # hexo需要的模块，不需要上传GitHub
      |- themes/        # 主题文件，需要上传GitHub的dev分支
      |- sources/       # 博文md文件，需要上传GitHub的dev分支，里面有个posts文件夹：放文章的。
      |- public/        # 生成的静态页面，由hexo deploy自动上传到gh-page分支
      |- package.json   # 记录hexo需要的包信息，不需要上传GitHub
      |- _config.yml    # 全局配置文件，需要上传GitHub的dev分支
      |- .gitignore     # hexo生成默认的.gitignore，它已经配置好了不需要上传的hexo文件

5.切换主题：
hexo默认主题为landscape，其他主题可在hexo官网选，下载主题好后，切记一定得看该主题的安装说明，有些主题需要安装插件，安装好后，建议执行下hexo clean命令

  1. 上传博客：
    post s -g   #生成网页，可本地预览
    post d -g    #生成网页，并部署到git上

    3种方式：
    第一种：自己已经拥有.md文件，只要把.md文件放到 source/_post文件夹下面，执行以上命令即可。
    第二种：post new “BlogName”,再执行以上命令。
    第三种：post new page “BlogName”,再执行以上命令。

  • Hexo 插入图片

    插入图片有3种方法，第一种是不用插件的情况，第二种是使用一种hexo的插件。第三种，上传图片生成图片链接软件：iPic 还可以生成MarkDown 格式的图片链接。

    但都需要同一个前提：修改 _config.yml 配置文件中的 post_asset_folder 项为 true.
    创建博客使用命令创建：hexo new [layout] “新博客名”，layout项可以省略。
    使用完命令之后，在source/_post文件夹里面就会出现一个“新博客名.md”的文件和一个“新博客名”的文件夹。下一步就是把需要的图片放到新创建的那个文件夹里面去。

  • Hexo 搬至云端

    hexo作为静态博客,缺点就是每次在更新博客时都要重新将每个静态页面重新生成一遍,再部署到页面托管的站点上,而且,一旦你不在自己的电脑旁,想要更新一下自己的博客就太困难了(除非你把博客项目放在网盘或是U盘上，但是ssh又要添加一条记录了).现在Coding的WebIDE 可以让你的hexo在云端编写，从此只要有网络，随时随地更新博客，也不用担心博客因为莫名其妙的操作而突然消失了!

使用方法也很简单，首先创建一个公有项目来存放你的hexo博客，然后进入WebIDE选项,Coding提供的在线集成开发环境，然后就可以通过terminal来搭建自己的云端写作环境，从此在云端写作了~

  • 遇到的一些问题
  1. 切换hexo主题时，显示不了博客，html显示不对

原因：未安装相关插件，缓存也未清理。
修改blog根目录的_config.yml,将theme修改为alpha

注意: 某些主题可能需要安装Node.js的插件,在安装主题时,最好在主题的Githu主页看看安装步骤。
安装插件命令： 

npm install hexo-renderer-插件名 --save
  1. 直接上传（.md）文章至hexo，未显示

  • 原因：.md 文件里面，最前面没有添加title，时间，tag 等字段，否则显示不了。

  • 解决方案：.md 文件里面，最前面需要添加title，时间，tag 等字段,注意可以使用下hexo clean 命令。
